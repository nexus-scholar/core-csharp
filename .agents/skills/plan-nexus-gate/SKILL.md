description: Plan a Nexus implementation gate before code changes; use for milestone decomposition, dependencies, risks, fixtures, and exit criteria.
---

Read `AGENTS.md`, `PLANS.md`, the target gate document, relevant ADRs, specifications, fixtures, and current code.

Produce:

1. Goal and scientific behavior delivered.
2. Inputs and source-of-truth references.
3. Dependency-ordered tasks with one module owner each.
4. Required fixtures and negative cases.
5. Allowed and excluded paths.
6. Risks, unresolved conflicts, and ADR needs.
7. Exact build, test, architecture, and conformance commands.
8. Measurable exit checklist.

Do not implement code. Stop when a source conflict would require guessing.
