Vision

Research software where authority is explicit

Nexus Scholar Core exists to make research workflows reconstructable. It treats protocols, evidence, amendments, automation, human decisions, and outputs as durable scientific state rather than incidental application data.

Operating belief

Automation can reduce friction, but it cannot become scientific authority. The ecosystem should help researchers move quickly while preserving the difference between a suggestion, a draft, an approved protocol, and a recorded decision.

Public position

Nexus is not trying to be another AI paper summarizer. The current project is an audit-oriented early-alpha, local-first research workflow kernel: strict Core records underneath, experimental research blocks on top, and AI assistance that remains proposal evidence until a human accepts it.

Project laws

  • Suggestion is not a decision.
  • Draft is not an approved protocol.
  • Current state is not historical record.
  • Vector retrieval is not an evidence source.
  • Automation is not scientific authority.

Current honesty

The project has implemented several Core contracts and UI foundations, but it is not a complete hosted review platform yet. The public site should make that visible: completed modules are documented, future work is named as future work, and unsupported claims stay out of the story.
