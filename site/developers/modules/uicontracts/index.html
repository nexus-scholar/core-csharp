Module documentation

UiContracts

Platform-neutral workspace and research-block contracts for renderers and app services that need a common display language without pulling UI frameworks into Core.

Scope and ownership

UiContracts defines the shared JSON-friendly model for future workspace renderers. The core records are WorkspacePlan, ResearchBlockDescriptor, EvidenceRef, ValidationRef, and BlockActionDescriptor, supported by enums for mode, severity, source kind, and action kind plus a small known-vocabulary surface.

This is not a scientific authority module. It does not decide protocol approval, deduplication acceptance, or screening outcomes. It exists so multiple renderers can describe the same work surface without each renderer inventing its own contract.

Boundary status

The authoritative source here is the implementation boundary document UI-CONTRACTS-v0, not an ADR. That is appropriate because the work is still renderer-oriented. The module is implemented as a contract library, and the repo already has tests and sample plans that consume it.

The important architectural guarantee is that NexusScholar.UiContracts has no project references. Core stays UI-free, and UI contracts stay framework-free.

Diagrams

Class map

The workspace contains blocks, and blocks carry evidence, validation, actions, and an optional JSON payload guarded as an object.

Use case flow

A trusted app service can compose a workspace plan once, then render it in desktop, CLI, or web surfaces without changing authority semantics.

Code surface and verification

The full contract lives in BlockContracts.cs. The code is intentionally direct: constructors guard blank ids and labels, payload JSON must be a valid object when present, known constants define early interoperable labels, and serializer options are standardized through UiContractJson.

The right way to think about this module is that it stabilizes display payloads before renderer code grows. The sample plans under samples/block-plans and the matching tests are integration aids, not scientific fixtures.

Related modules and current risks

UiContracts is consumed by Avalonia Blocks today and could be consumed by CLI or Web renderers later. Its main risk is accidental authority inflation: once a block shows a decision or warning, it becomes tempting to treat the block itself as canonical state. That would be a mistake.

The current design pushes in the right direction. Evidence refs point outward, actions declare intent, and the contract never claims to be the domain record itself.
