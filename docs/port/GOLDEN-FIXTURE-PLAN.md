# Golden Fixture Plan

Status: Gate 0 planning only. No fixtures are generated by this document.

## Rules

- Generate fixtures only from the PHP commit pinned in `specs/SOURCE.lock.json` when the source is PHP behavior.
- Never hand-edit generated outputs.
- Record source commit, generator command, generator version, input digest, and output digest for every generated fixture set.
- Treat a C# test failure as a defect or a conflict signal, not permission to replace the fixture.
- Classify every difference as one of:
  - equivalent serialization,
  - intentional change,
  - PHP defect,
  - C# defect,
  - unresolved specification conflict.

## Storage Plan

- Blueprint and contract fixtures: `fixtures/conformance/`
- PHP-generated differential fixtures: `fixtures/php-golden/`
- Each fixture set should include a small manifest file with:
  - fixture id,
  - source kind,
  - source refs,
  - source commit,
  - generator command,
  - generator version,
  - input digest,
  - output digest,
  - semantic comparison rules.

## Comparison Rules

- Ignore only documented serialization differences such as object key order when canonicalization is explicitly fixed.
- Do not ignore semantic fields.
- Ignore generated ids or timestamps only when the fixture harness cannot inject fixed values.
- Compare normalized identifiers, digest inputs, approval semantics, snapshot membership, provenance meaning, and audit metadata.

## Generation Order

1. Freeze blueprint contract fixtures for kernel, protocol, workflow, artifacts, provenance, bundles, extensibility, and AI governance.
2. Add a PHP exporter or equivalent generator harness for shared identity, search, deduplication, corpus lock, screening, citation network, full-text, and exports.
3. Build semantic comparators before implementation claims.
4. Only after fixtures exist should implementation gates claim compatibility.

## Current Gaps

- `fixtures/conformance/protocol-minimal.json` is hand-written and lacks generator metadata.
- No PHP-generated differential fixtures exist yet.
- No semantic comparator report format exists yet.
- The blueprint pack declares conformance intent but does not ship the actual fixture assets needed by this repo.
