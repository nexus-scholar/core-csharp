Module documentation

Kernel

Deterministic primitives and canonicalization contracts that every higher-level module uses for authoritative identity and reproducibility.

Scope and ownership

Kernel defines canonical IDs, value objects, canonical JSON representation rules, UTC timestamp format, and digest-envelope behavior. It is a domain-only module and must remain dependency-free toward applications and infrastructure.

  • Core primitives: IClock, EntityId, Guard, digest vocabulary, canonical timestamp policy.
  • Canonical contracts: CanonicalJson* model + serializer, normalized Unicode handling, and array/object shape conventions.
  • Digest contracts: ContentDigest, DigestEnvelope, DigestScope, DigestAlgorithm.

Gate and ADR status

Gate 2 is the currently accepted authority scope for deterministic kernel behavior. Source decision is ADR 0002.

  • Canonical digest format: sha256:<64 lowercase hex>.
  • Canonical JSON profile: rfc8785-jcs with deterministic property ordering.
  • Canonical timestamp format: yyyy-MM-ddTHH:mm:ss.fffffffZ.
  • Digest scope vocabulary includes raw-artifact, canonical-json, protocol-content, approval-record, provenance-event, bundle-manifest, and ndjson-stream.
  • Digest envelope validator excludes draft/local defaults and requires schema + version fields.

Evidence file set: GATE-02-EVIDENCE.

Current issues to track

  • Protocol still carries legacy provisional digest material in parts of its implementation. The local gate intentionally limits scope to kernel determinism and does not upgrade protocol compatibility.
  • Non-integer very large numeric literals. CanonicalJson.ParseNumber accepts long integers and decimal/double via explicit parse paths; extreme numeric inputs should be validated at schema boundaries if precision assumptions matter.
